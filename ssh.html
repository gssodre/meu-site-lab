Aqui eu registro tudo que estou aprendendo sobre acesso remoto via SSH: conexão, autenticação, máquina de origem, máquina de destino e erros comuns.

O que é SSH

SSH é um jeito de entrar remotamente em outra máquina pelo terminal. No meu laboratório, eu uso isso para acessar o Ubuntu Server a partir do Mint ou do Debian.

Comando principal

Máquina: Linux Mint Host ou Debian Cliente

ssh ubuntu@192.168.56.12

O que deve aparecer:

Se der certo, o terminal muda de algo como mint@host para ubuntu@servidor.

Como eu penso antes de usar SSH

  • Primeiro confirmo o IP do servidor
  • Depois confirmo se estou na máquina certa
  • Depois testo a comunicação da rede
  • Só então tento o acesso SSH

Comandos úteis

Entrar no servidor

Máquina: Mint Host ou Debian Cliente

ssh ubuntu@192.168.56.12

Ver se o serviço SSH está ativo

Máquina: Ubuntu Server

sudo systemctl status ssh

O que deve aparecer: algo como active (running).

Ver tentativas e acessos no log

Máquina: Ubuntu Server

sudo tail -f /var/log/auth.log

O que deve aparecer: registros de login, chave pública aceita, desconexão ou falha de autenticação.

Erros comuns

No route to host

A máquina não conseguiu alcançar o IP do servidor.

Connection refused

A máquina alcançou o servidor, mas o serviço SSH não estava ouvindo na porta 22.

Permission denied

O IP e o serviço estavam certos, mas a autenticação falhou.

Rodar o comando na máquina errada

Antes de digitar qualquer comando, eu preciso olhar o prompt e confirmar onde estou.

Teste real de autenticação SSH com usuário inválido

Fiz um teste tentando acessar o servidor com um usuário que não existia no sistema, para observar o comportamento do SSH e depois analisar os logs.

Comando usado

Máquina: Linux Mint Host

ssh ubuntu-teste@192.168.56.12

O que apareceu no terminal

mint@host:~$ ssh ubuntu-teste@192.168.56.12
ubuntu-teste@192.168.56.12's password:
Permission denied, please try again.
ubuntu-teste@192.168.56.12's password:
12
Permission denied, please try again.
ubuntu-teste@192.168.56.12's password:
12ubuntu-teste@192.168.56.12: Permission denied (publickey,password).

O que aconteceu

O acesso falhou porque tentei autenticar com o usuário ubuntu-teste, mas esse usuário não existe.

O que aprendi

  • SSH pode falhar por senha errada ou por usuário inexistente
  • Mesmo sem login bem-sucedido, o sistema registra a tentativa
  • O log mostra de qual IP a tentativa veio

Interpretação do log

  • Invalid user: o usuário não existe
  • Failed password: houve tentativa de autenticação
  • rhost=192.168.56.1: a tentativa veio do Mint Host
  • Connection closed: a sessão foi encerrada sem sucesso

Resumo do que aprendi

SSH não é só “entrar na máquina”. Ele também ensina rede, autenticação, serviço ativo, logs e visão de administração real de servidor.
