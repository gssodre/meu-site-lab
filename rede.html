Objetivo desta página

Aqui eu registro tudo que estou aprendendo sobre rede no meu laboratório: IP, interface, Host-Only, NAT, comunicação entre máquinas e testes básicos.

Máquinas do laboratório

  • Linux Mint Host: máquina principal
  • Ubuntu Server: servidor do laboratório
  • Debian Cliente: máquina de testes

Tipos de rede que usei

Host-Only

Rede interna entre o host e as VMs. Serve para as máquinas se comunicarem dentro do laboratório.

NAT

Serve para a VM ter acesso à internet. Normalmente não é usada para uma VM conversar diretamente com outra no laboratório.

Comandos importantes

Ver interfaces e IPs

Máquina: qualquer máquina Linux

ip a
hostname -I

O que deve aparecer: nome da interface e o endereço IP da máquina.

Ver rota padrão

Máquina: qualquer máquina Linux

ip route

O que deve aparecer: gateway padrão e rede usada pela máquina.

Testar comunicação com o servidor

Máquina: Mint Host ou Debian Cliente

ping 192.168.56.13

O que deve aparecer: respostas com tempo em ms, mostrando que a máquina alcançou o servidor.

Como eu penso na rede do laboratório

  • Primeiro descubro o IP da máquina
  • Depois vejo em qual interface esse IP está
  • Depois testo comunicação com ping
  • Se não comunicar, eu verifico IP, rota, interface e tipo de rede no VirtualBox

Erros comuns que já aconteceram

No route to host

Significa que a máquina não conseguiu alcançar o IP informado.

Conexão recusada

Significa que a máquina foi alcançada, mas não havia serviço ouvindo naquela porta.

Rodar comando na máquina errada

Antes de rodar qualquer comando, eu preciso conferir se estou no mint@host ou no ubuntu@servidor.

Resumo do que aprendi

Sem rede funcionando, não existe laboratório. Por isso IP, interface, rota e conectividade são a base de tudo que estou fazendo.
